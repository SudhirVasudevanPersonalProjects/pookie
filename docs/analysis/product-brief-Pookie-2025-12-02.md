# Product Brief: Pookie

**Date:** 2025-12-02
**Author:** sudy

**Purpose:** Internal planning document for development. This is my personal project - built for me, by me - to solve my scattered thoughts problem. Not a business venture, just a portfolio piece that proves I can build production ML systems.

---

## Executive Summary

I built Pookie because I had a problem that no existing tool could solve: my brain dumps ideas everywhere, and I lose them in the chaos.

**The Problem I Faced:**
Screenshots I never look at. Notes that make sense in the moment but are gibberish later. Long rambling paragraphs with 5 different thoughts mixed together. Bookmarks, links, half-formed ideas scattered across a dozen apps. I tried productivity tools - they made it worse. I tried journaling apps - they wanted perfect prose. I tried note-taking apps - they gave me folders and tags but no *understanding*.

**What I Built:**
Pookie is my personalized LLM - an AI that learns how MY mind works, understands what I care about, and finds patterns I can't see. It's my external brain that takes chaotic thought dumps and makes them useful.

**How It Works:**
- **Call Agents Mode:** I paste rambling paragraphs → AI automatically separates distinct thoughts → I chat to refine
- **Semantic Clustering:** Vector embeddings group related content into "abodes" without manual organization
- **Discover Mode:** AI learns my taste profile and recommends new things I'd actually care about
- **RAG Pipeline:** Chat interface that knows my context, not generic ChatGPT responses
- **Pookie-verse:** Knowledge graph visualizing how my thoughts connect

**Why It Matters (For My Career):**
This demonstrates end-to-end ML system design: RAG pipeline, multi-agent orchestration, vector search, embeddings - all on free tiers (~$0-3/month) to prove technical efficiency. It's not a tech demo; it's solving my real problem with production-grade architecture.

## MVP Scope

### Core Features (Build All, Keep Simple)

I'm building ALL the core features from my resume, implemented in the simplest way possible using free-tier architecture.

**1. Capture (with AI-Powered Thought Separation)**
- Text input box for brain dumps
- Mic button → speech-to-text (iOS native voice recognition)
- **Split button for save:**
  - "Save" → saves as-is
  - "Separate & Save" → AI splits distinct thoughts first, then saves each separately
- Backend: OpenRouter free models for thought separation logic
- Storage: Supabase PostgreSQL

**2. Abodes (Semantic Clustering)**
- **Embeddings:** sentence-transformers (local, free) running on FastAPI backend
- **Vector DB:** FAISS (local file storage, free)
- **Clustering:** Automatic grouping into 3-5 thematic abodes based on semantic similarity
- **Naming:** LLM generates abode names based on content
- **UI:** Simple list view of abodes with care counts
- **Knowledge graph structure exists** (abodes + relationships) - just no visual UI yet

**3. Discover Mode (Action Suggestions)**
- **Learn taste profile** from what I save in my abodes
- **AI recommends actions** based on my cares:
  - Content actions: "Watch this movie" "Listen to this song"
  - Goal actions: "You care about getting jacked → Hit the gym"
- **UI:** One simple suggestion at a time - "Do this next"
- **Backend:** Embedding similarity + LLM synthesis via Claude Haiku

**4. RAG Chat (My Personal LLM)**
- Ask questions about my cares: "What did I say about X?"
- **RAG pipeline:**
  - FAISS vector search across all my cares
  - Retrieve top-k relevant cares
  - LLM response with context (Claude Haiku)
- **UI:** Simple chat interface
- Proves end-to-end RAG implementation
